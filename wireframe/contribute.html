Contribute an entry

The catalog is a data-only GitHub repo. You add a scheme or implementation by opening a PR with one metadata file — no site code. Benchmarks run on a hidden test set we host, so you submit the implementation, never the numbers.

1
Fork the data repo

One file per entry, keyed by a stable id like impl/my-zorch.

2
Fill the schema

Pick a category and fill its fields. Abstract primitives stay field-agnostic; implementations fill every slot. See schema.md.

3
Open a PR

CI validates the file against the schema and resolves your uses slots to existing entries.

4
We benchmark & merge

On merge, your entry runs against the hidden profile set and lands on the leaderboard.

Slots (field, hash, pcs, …) reference existing scheme ids. Missing a primitive? Add it as its own PR first.

Stays private (server-side).
  • TestInput — ELFs / hash vectors (DDoS & large-payload risk)
  • SRS — one canonical ceremony, referenced by hash
  • the hidden benchmark vectors

You never submit these — that's what keeps the leaderboard fair and the service safe.
